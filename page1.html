Program Structure

The idea is that I upload a structured and almost fully developed script onto Github and then just "git clone" it to the mini-PC. Below I'll put the link to the Github project, but note that it is only the first version of the project and I'll certainly have to change things about it locally on the mini-PC, so the project on Github is not functional in itself.

https://github.com/lirinton/document-scanner

The structure of the program is depicted in the following tree on Github:

Here are the dependencies being used in the project, these are by far not the only open-source materials being used, but these are the main dependencies:

flask==2.3.3
opencv-python==4.8.1.78
pillow==10.0.1
pytesseract==0.3.10
numpy==1.24.3

Yesterday I went ahead and downloaded Ubuntu onto the mini-PC and installed everything and even cloned the repository onto it. It was a difficult choice between installing the server version and the desktop version, but finally I decided to go with the desktop version as it makes everything easier to access if SSH were to stop working. However, I have not yet started the implementation process.

If during the process I use a different dependency that is part of the main structure of the project, then I will upload that as well to this blog. With this, in the next blog entry I will start setting up the mini-PC.
